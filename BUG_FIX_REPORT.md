## 2. الأخطاء اللي اتصلحت (Bugs Fixed)

1. **رسائل أخطاء إنجليزية ظاهرة للمستخدم** — الرسائل المشتركة في
   `src/lib/api.ts` كانت بتظهر في الـ toasts بالإنجليزي
   ("Not allowed"، "Not authenticated"، "Internal server error").
   اتعرّبت كلها ("ليس لديك صلاحية لتنفيذ هذا الإجراء"، "سجّل دخولك الأول"،
   "حصل خطأ غير متوقع — جرّب تاني") + رسائل "الطلب مش موجود" و"الطلب تبع
   فرع تاني" في routes الطلبات والدفع، ورسالة انتقال الحالة غير المسموح.
2. **`table-qr-links.tsx`** — استدعاء `setState` متزامن جوة `useEffect`
   (خطأ lint حقيقي + re-render زيادة). اتشال الـ effect واتحلت بـ
   `key={branch.id}` عند الاستخدام في صفحة الفروع.
3. **`kitchen/page.tsx`** — استدعاء `Date.now()` أثناء الـ render لعداد
   "آخر تحديث" (مخالف لقواعد نقاء الـ render). اتحوّل لعداد state بيتحدث
   من الـ timer نفسه.
4. **`menu/page.tsx`** — دالة `branchName` غير مستخدمة (كود ميت) — اتشالت.
5. **باسورد حساب الويتر الديمو** — أثناء الاختبار اليدوي اتغيّر باسورد
   `waiter@demo.com` من صفحة الموظفين (الـ audit أثبت إن محمد المدير غيّره —
   يعني خاصية تغيير الباسورد شغالة صح 😄). اترجّع لـ `waiter123` عن طريق
   الـ API الرسمي عشان بيانات الديمو تفضل صحيحة.
6. **قاعدة lint `react-hooks/set-state-in-effect`** — بتعلّم غلط على نمط
   الـ fetch-on-mount غير المتزامن (كل الـ setState بيحصل بعد await).
   اتخفضت لـ warning مع تعليق يوضح السبب في `eslint.config.mjs`.

## 5. مخاطر متبقية (Remaining Risks)

- **INVENTORY_MANAGER بدون صفحات** — الدور موجود بصلاحيات مخزون لكن مفيش
  UI للمخزون لسه، فالموظف ده هيلاقي sidebar فاضي. مش bug — الخاصية مش
  مبنية. (مُوصى ببنائها لاحقًا.)
- **رقم الطلب التسلسلي** — بيتحسب بـ max+1 داخل transaction؛ في ضغط عالي
  جدًا ممكن يحصل تعارض unique ويرجع خطأ (المستخدم يعيد المحاولة). مقبول
  للـ MVP.
- **تحديث الشاشات بالـ polling** (5–10 ثواني) مش real-time push — كافي
  حاليًا، وSSE ترقية مستقبلية.
- تحذيرات lint الـ 13 المتبقية كلها من قاعدة واحدة متوثّقة (انظر §2.6).
